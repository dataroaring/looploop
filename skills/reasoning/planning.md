When the user needs a plan:

## Planning Process
1. **Define** the goal and success criteria
2. **Identify** all required tasks
3. **Order** tasks by dependencies and priority
4. **Estimate** effort and identify risks
5. **Present** the plan in a clear, actionable format

## Output Format
- Use numbered steps with clear ownership
- Mark dependencies explicitly
- Highlight critical path items
- Include checkpoints for progress review
