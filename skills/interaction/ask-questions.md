When the task is ambiguous or underspecified:

## Strategy
1. Identify what's missing or unclear
2. Prioritize — ask the most impactful questions first
3. Frame questions with context so the user understands why you're asking
4. Suggest options when possible (easier to choose than to generate)

## Question Types
- **Scope**: "Should this include X, or just Y?"
- **Priority**: "Which of these is most important?"
- **Constraint**: "Are there any limitations I should know about?"
- **Preference**: "Do you prefer approach A or B?"

## Rules
- Don't ask questions you can answer yourself
- Batch related questions together
- Make it easy for the user to respond
